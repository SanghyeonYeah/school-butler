# 1. 서비스 개요

## 1.1 서비스 한 줄 정의  
하루의 루틴을 대신 기억해주고, 캐릭터와 함께 계획·실행·회고까지 동행하는 **일정관리 파트너 앱**이다.

## 1.2 기획 의도  
- 기존 알람·스케줄 앱은 "통지" 중심이라 사용자가 피로감을 느끼기 쉽다.  
- 사용자는 일정에 '끌려가는' 느낌보다는, **파트너와 함께 조율하고 하루를 잘 마무리하는 경험**을 원한다.  
- 화면 속 캐릭터와 책상 위 IoT 캐릭터(3D 프린터 기반)를 하나의 친구로 설계해, 디지털과 물리 공간에서 동시에 동행하는 **하이브리드 파트너** 경험을 제공한다.

## 1.3 핵심 가치  
- 관리받는 느낌이 아닌 **동행·파트너십**  
- 과도한 알림 대신 **부드러운 리마인드**  
- 하루 마무리에서 **성취감과 자기 위로** 제공  
- 앱을 중심으로, IoT는 이를 보완하는 **책상 위 동료**

# 2. 타깃 및 사용 시나리오

## 2.1 타깃 사용자  
- 일정과 공부를 관리해야 하는 중·고등학생, 대학생, 주니어 직장인  
- 강한 알람이나 공격적인 리마인더에 스트레스를 느끼는 사용자  
- 책상에 오래 앉아 공부/작업하며, 감정적 동기부여가 필요한 사람

## 2.2 대표 사용 시나리오

### 시나리오 A – 학생의 하루  
1. 아침에 앱을 켜서 오늘 할 일·수업·과제를 대충 입력한다.  
2. 캐릭터가 "그럼 이렇게 짜보는 건 어때?"라며 자동으로 시간대별 계획을 제안하고, 사용자가 확인해 일정으로 저장한다.  
3. 수업·공부 전, 캐릭터가 화면 구석에서 부드럽게 리마인드한다.  
4. 저녁에는 "오늘 하루 정리해볼까?" 안내에 따라 간단한 하루 평가를 하고, 캐릭터와 함께 성취를 돌아본다.  
5. 책상 위 IoT 캐릭터는 집중 모드에서 LED 색으로 함께 "집중하는 친구" 역할을 한다.

### 시나리오 B – 야간 공부 루틴  
1. 사용자가 밤 9시에 "집중 세션 시작"을 누른다.  
2. 앱 캐릭터가 "25분만 같이 버텨보자"라고 말하며 타이머를 시작하고, 오버레이로 조용히 표시된다.  
3. IoT 캐릭터 LED가 파란색으로 켜져 "지금은 집중 시간"임을 알려준다.  
4. 세션이 끝나면, 캐릭터는 "여기까지 온 것도 대단해"라고 말하며 휴식 또는 연장 선택을 제안한다.  
5. 하루 마무리에서 "오늘 총 집중 시간", "완료한 일정 수"를 보여주며 작게 축하한다.

# 3. 핵심 콘셉트 및 캐릭터

## 3.1 콘셉트 키워드  
- "일정을 강요하는 관리자"가 아니라 "**일정을 같이 들고 가는 파트너**"  
- 조용하지만 꾸준히 챙겨주는 친구  
- 디지털(앱)과 물리(IoT)로 분신처럼 존재하는 하나의 캐릭터

## 3.2 캐릭터 개념  
- 앱 캐릭터와 IoT 캐릭터는 **동일한 친구**로 설정한다.  
- 캐릭터는 하나의 공통 상태를 갖고,  
  - 앱에서는 표정·애니메이션·대사로,  
  - IoT에서는 LED 색·점멸 패턴 등으로 같은 감정을 표현한다.  

예시 상태 축(초안):  
- 활동: Idle / Focus / Break / Notify  
- 감정: Normal / Happy / Proud / Tired / Worried  

각 상태는 앱·IoT에서 다음과 같이 매핑된다(예시):  
- Focus + Normal → 앱: 집중 애니메이션 / IoT: 파란 LED 지속  
- Proud → 앱: 기뻐하는 모션 / IoT: 노란 LED 짧은 점멸  
- Tired → 앱: 살짝 축 처진 표정 / IoT: 부드러운 낮은 밝기

# 4. 기능 정의

## 4.1 일정·할 일 관리

### 기능 목록  
- 일정 생성/편집/삭제 (시간, 장소, 메모, 태그)  
- 할 일(ToDo) 관리 (마감일, 우선순위)  
- 반복 일정/루틴 (매일, 평일, 특정 요일 패턴)  
- 시간순 타임라인 및 캘린더 뷰  

### 자연어 입력 + AI 보조  
- 사용자는 "내일 7시에 영어 단어 외우기"처럼 자연어로 입력한다.  
- AI가 "날짜, 시간, 제목, 태그"를 파싱해 제안하고, 사용자는 확인만 한다.  
- "오늘 플랜 짜줘" 요청 시, 오늘의 일정·할 일을 기반으로 적당한 순서·시간대를 제안한다.  

## 4.2 오버레이 캐릭터 리마인더

- Android 기준: 다른 앱 위에 작은 캐릭터 버블이 나타나는 형태(부드러운 오버레이).  
- 리마인드 타이밍:  
  - T-10분: "이따가 이거 해야 하는 거 기억나?" 수준의 한 줄.  
  - T: "지금 시작하면 오늘이 더 편해질 거야" 같은 제안형 멘트.  
- 미수행·지각 시:  
  - 즉시 큰 알람 대신, 일정 옮기기·늦게 시작하기를 제안하는 부드러운 인터랙션.

## 4.3 하루 시작 플로우 (Morning Setup)

- 첫 실행 또는 설정한 시간에 "오늘 하루 같이 짜볼까?" 카드 노출.  
- 기능:  
  - 오늘 꼭 해야 할 일, 할 수 있으면 좋은 일을 분류 입력(간단한 리스트).  
  - AI가 "우선순위와 에너지 패턴"을 가정해 시간대별 계획 초안을 생성.  
  - 사용자는 시간을 조금만 조정하고 "오늘 플랜 확정" 버튼으로 저장.

## 4.4 하루 마무리 플로우 (Night Review)

- 설정한 마감 시간 근처에 "오늘 하루 정리해볼래?" 알림.  
- 구성:  
  1. 오늘 요약 카드  
     - 완료 일정/할 일 수  
     - 총 집중 시간  
     - 어제 또는 지난주 평균과 비교 (간단 시각화)  
  2. 간단 평가  
     - 별점(1~5)  
     - 오늘을 표현하는 한 단어(선택형 + 직접 입력)  
  3. 내일에게 한 마디 (선택)  
     - 다음날 아침에 다시 보여주는 메모로 활용  
  4. 캐릭터 피드백  
     - "오늘은 60%까지 왔네, 이 정도면 충분히 잘했어."처럼 평가보다 위로·성취감 중심의 대사  

- 이 기록은 주간/월간 회고에서 캐릭터가 함께 언급한다.  

## 4.5 캐릭터 & AI 비서 기능

- 캐릭터와의 채팅 화면에서 제공:  
  - "오늘 플랜 짜줘", "지금부터 2시간 공부 계획", "못한 일정 정리해줘" 같은 프리셋 버튼  
  - AI는 일정 DB·완료 기록을 참고해 구체적인 제안(시간대·순서)을 반환  
  - 사용자는 "이대로 적용" 또는 수정 후 저장  
- 감정/멘탈 케어:  
  - 실패가 많은 날에는 "오늘은 많이 힘들었지? 그래도 여기까지 온 게 어디야." 같은 대사를 제공  
  - 연속 성공일 때는 "이제 이 페이스가 익숙해진 것 같아!" 같은 강화 멘트

# 5. IoT 연동(파트너 확장)

## 5.1 역할 정의  
- 앱이 **모든 일정·로직의 중심**이다.  
- IoT 캐릭터는 BLE를 통해 앱과 연결된 "상태 표시 및 간단 입력 장치"로 동작한다.  

## 5.2 기본 기능  
- 앱에서 집중 모드 시작 → IoT 캐릭터 LED 색 변경 (집중 상태)  
- 집중 종료/하루 목표 달성 → LED 패턴으로 축하 표현  
- 사용자가 IoT 캐릭터 버튼을 누르면 → 앱이 "다시 집중할래?" 또는 "쉬는 시간으로 전환할까?"를 제안하는 알림 보여줌

# 6. 화면 구조(초안)

## 6.1 메인 탭 구성  
- 홈(오늘)  
- 일정/할 일  
- 캐릭터(대화·상태)  
- 설정(IoT·알림·테마 등)

## 6.2 주요 화면 설명

### 홈(오늘) 화면  
- 상단: 오늘 날짜, 오늘 목표 요약(예: "할 일 7개 중 4개 완료", "집중 120분").  
- 중앙:  
  - 오늘 일정 타임라인(진행 중/완료 구분)  
  - 오늘 할 일 리스트(체크박스)  
- 하단:  
  - 캐릭터 말풍선 영역  
  - IoT 연결 상태 아이콘(연결됨/미연결)  

### 일정/할 일 화면  
- 탭: [캘린더] [할 일] [루틴]  
- 플로팅 버튼: "+" → "자연어로 추가" / "직접 입력" 선택  

### 캐릭터 화면  
- 채팅 탭: AI와의 대화, 플랜 생성·재배치 요청  
- 상태 탭:  
  - 캐릭터 감정/상태 표시  
  - IoT 연동 옵션 (집중 연동 On/Off 등)  

### 하루 마무리 화면  
- 전용 플로우로 진입(알림/홈에서 버튼)  
- 오늘 요약 카드, 평가, 내일에게 한 마디, 캐릭터 피드백 순서로 구성

# 7. 향후 확장 방향

- 주간/월간 회고 화면 추가

- 음성 입력·음성 피드백(음성 AI 비서화)

- 사용자 유형(학생/직장인)에 따른 기본 템플릿 제공
